Changelog

Release notes for VERA — the OCR validation platform.

1.5.0 - 2026-04-02

Added

  • Invoice export — validated documents can be exported in five formats: json, txt, csv, facturx (EN 16931 CII), ubl (UBL 2.1 / PEPPOL)
  • Missing EN 16931 fields reported in X-VERA-Warnings response header
  • LLM routes — AI summaries, model listing, health check, model pull (sync + streaming)
  • Frontend auth components — AuthProvider, AppHeader, login page with CSRF and network-resilient error handling
  • 51 new invoice export tests; 44 frontend component tests; 150 backend tests total

Changed

  • API routes refactored into dedicated modules: export.py, license.py, llm.py
  • Line items stored as JSON in structured fields for structured invoice export

Dependencies

  • lxml>=5.2.0 added for Factur-X and UBL XML generation

1.4.0 - 2026-04-01

Security

  • Session cookie secure=true by default (configurable via SECURE_COOKIES)
  • HubUnavailableError — Hub outage at login now returns 503 instead of 401

Added

  • License enforcement middleware — graduated enforcement: grace, licensed, warning, soft (uploads blocked), hard (all blocked)
  • GET /api/license/status proxy endpoint
  • Celery Beat service for stuck-task recovery
  • Worker Docker healthcheck via celery inspect ping
  • Credential cache — HMAC-keyed Redis cache (4-hour TTL) for Hub outage resilience
  • Validation status guard — returns 409 if document not in reviewable state

Changed

  • Cold-cache enforcement default changed from grace to soft (read-only)
  • Soft mode now blocks validation and summary endpoints (previously uploads only)

Fixed

  • Default logging changed from DEBUG to INFO
  • SSE status stream now terminates at terminal statuses
  • Re-export bug for documents in exported status
  • Summary status ordering — reverts to validated on LLM failure
  • Login page layout and frontend TypeScript build fixes

1.3.0 - 2026-03-18

Security

  • CSRF enforcement — validates X-CSRF-Token header on all mutating requests
  • Redis-backed sessions — replaced in-memory dict with Redis
  • Production frontend build — multi-stage Dockerfile with Next.js standalone output

Added

  • License check — queries Hub's /api/license/status on startup
  • GET /api/csrf-token endpoint
  • Premium visual polish: self-hosted Inter font, micro-interactions, skeleton loaders, illustrated empty states, custom scrollbars, branded footer

1.2.0 - 2026-03-06

Security

  • Fixed CSV injection vulnerability — uses csv.writer with QUOTE_ALL
  • Fixed command injection in virus scan hook
  • Security headers on all responses; CORS tightened to explicit whitelists
  • Containers run as non-root users

Added

  • Healthchecks on all services
  • Celery task time limits (600s hard, 540s soft)

1.1.0 - 2026-03-03

Changed

  • Ollama connectivity switched to shared ollama_network Docker network
  • Redis AOF persistence; Celery task_acks_late and worker_prefetch_multiplier=1
  • Resource limits on all services; auto-restart policy

Added

  • Scheduled backup service with retention cleanup
  • Stuck-document recovery via Celery Beat

1.0.0 - 2026-02-06

Added

  • Initial stable release of VERA
  • JPG, PNG, and multi-page PDF uploads with PaddleOCR local inference
  • Confidence-based token highlighting and inline correction UI
  • Validation hard gate — exports only unlock after explicit review
  • PostgreSQL + Alembic migrations; Celery + Redis async worker
  • SSE status streaming, Prometheus metrics, document retention policy
